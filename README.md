# Claude API Test

Small Jupyter notebooks for learning the Anthropic Python SDK, covering multi-turn conversations, prompt evaluation, tool use, streaming, built-in tools, and a full RAG retrieval pipeline.

## Project Structure

- `multi_turn_chat_example.ipynb` - a simple example that sends an initial prompt, stores Claude's reply, and sends a follow-up message with conversation history.
- `chatbot.ipynb` - an interactive chatbot loop that keeps the message history, skips empty input, handles API errors, and exits when you type `quit`.
- `001_prompt_evals.ipynb` - the starting point for a prompt evaluation pipeline: generates an AWS-related task dataset, runs each task through Claude, and collects results (grading stubbed).
- `001_prompt_evals_fns.ipynb` - extends the base eval with a `format` field per task, syntax validation (`validate_json`, `validate_python`, `validate_regex`), model-based grading via `grade_by_model`, and a combined score averaged from both grades.
- `001_prompt_evals_grader.ipynb` - adds a `solution_criteria` field to each dataset item and passes it to the grader, producing richer results with `strengths`, `weaknesses`, and `reasoning` alongside the score.
- `001_prompt_evals_complete.ipynb` - the fully assembled version combining all features: criteria-aware grading, syntax validation, combined scoring, strengths/weaknesses output, and average score reporting.
- `001_prompting.ipynb` - introduces the `PromptEvaluator` class for automated prompt testing with concurrent execution, HTML report generation, and JSON output; uses an athlete meal plan scenario.
- `002_prompting_completed.ipynb` - same as `001_prompting.ipynb` with fully executed cell outputs demonstrating the evaluation results.
- `001_tools.ipynb` - introduces tool use: defines `get_current_datetime` and `add_duration_to_datetime` as `ToolParam` schemas and walks through the manual tool-calling flow step by step.
- `001_tools_007.ipynb` - tool use with `Message`-aware helper functions that accept either a raw string or an `anthropic.types.Message` as content.
- `001_tools_008.ipynb` - adds a `run_tool` dispatcher and `run_tools` helper to automatically execute all tool calls in a response; single-turn tool use example.
- `001_tools_009.ipynb` - full multi-tool agentic loop using `run_conversation` that keeps calling Claude until no more tool use is requested; demonstrates multi-step date arithmetic reasoning.
- `003_tool_streaming.ipynb` - streaming tool use with a `save_article` tool; introduces `chat_stream` with fine-grained event handling and a `run_conversation` loop for streamed multi-turn tool calls.
- `003_tool_streaming_completed.ipynb` - same as `003_tool_streaming.ipynb` with fully executed streaming outputs.
- `005_text_editor_tool.ipynb` - implements `TextEditorTool` wrapping Claude's built-in `str_replace_based_edit_tool`; supports `view`, `create`, `str_replace`, `insert`, and `undo_edit` commands with automatic backups.
- `006_web_search.ipynb` - starting point for using Claude's built-in web search tool; sets up the client and message helpers with an empty prompt ready to fill in.
- `006_web_search_complete.ipynb` - complete web search example using the `web_search_20250305` schema with a 5-use cap and domain filtering restricted to `nih.gov`; answers a health-related query.
- `001_chunking.ipynb` - text chunking strategies: fixed-size character chunks with overlap, sentence-level chunks with overlap, and section-level chunks by markdown headers.
- `002_embeddings.ipynb` - generates embeddings with the VoyageAI `voyage-3-large` model via section-based chunking on `report.md`.
- `003_vectordb.ipynb` - implements a `VectorIndex` class with cosine similarity and walks through the full dense RAG pipeline: chunk → embed → store → query → retrieve top-k results.
- `004_bm25.ipynb` - implements a `BM25Index` class (k1/b configurable) for sparse keyword retrieval over section-chunked documents.
- `005_hybrid.ipynb` - combines `VectorIndex` and `BM25Index` inside a `Retriever` class for hybrid dense+sparse search with score fusion.
- `dataset.json` - the generated evaluation dataset of three AWS-related tasks (Python, JSON, and Regex formats) used by the eval notebooks.
- `report.md` - synthetic interdisciplinary research report used as the sample document corpus for the RAG notebooks.
- `output.html` - HTML evaluation report generated by `001_prompting.ipynb` / `002_prompting_completed.ipynb`.
- `output.json` - JSON grading results produced by the prompt evaluation pipeline.
- `main.py` - minimal placeholder Python script.
- `.env` - local environment variables such as `ANTHROPIC_API_KEY` and `VOYAGE_API_KEY`.
- `.venv/` - local Python virtual environment.

## Requirements

- Python 3.11+
- An Anthropic API key
- A VoyageAI API key (for the RAG notebooks: `002_embeddings.ipynb`, `003_vectordb.ipynb`, `005_hybrid.ipynb`)

## Setup

1. Create and activate a virtual environment.
2. Install dependencies:

```bash
pip install anthropic python-dotenv voyageai
```

3. Create a `.env` file in the project root:

```env
ANTHROPIC_API_KEY=your_api_key_here
VOYAGE_API_KEY=your_voyage_api_key_here
```

## How To Use

Open any notebook in VS Code or Jupyter and run the cells from top to bottom.

### `multi_turn_chat_example.ipynb`

This notebook shows the basic message flow:

- start with an empty `messages` list
- add a user message
- call Claude
- add the assistant reply back into the list
- send a follow-up message with full context

### `chatbot.ipynb`

This notebook turns the same helper functions into a looped chatbot:

- prompts for user input
- stores each user message
- sends the full message history to Claude
- stores Claude's reply
- prints the reply
- exits when you type `quit`

### `001_prompt_evals.ipynb`

This notebook introduces the prompt evaluation pipeline:

- generates a dataset of AWS-related tasks via Claude and saves it to `dataset.json`
- loads the dataset and runs each task through Claude
- collects outputs into a results list (grading is left as a stub)

### `001_prompt_evals_fns.ipynb`

This notebook extends the base eval with grading functions:

- generates a dataset where each task includes a `format` field (`python`, `json`, or `regex`)
- uses prefill and stop sequences to elicit clean code-only responses from Claude
- validates syntax automatically using `ast.parse`, `json.loads`, or `re.compile`
- grades each output with a second Claude call (`grade_by_model`) and combines both scores
- prints the average score across all results

### `001_prompt_evals_grader.ipynb`

This notebook adds criteria-aware grading:

- dataset items include a `solution_criteria` field describing what a good answer looks like
- the grader receives the criteria and returns structured feedback with `strengths`, `weaknesses`, `reasoning`, and a numeric `score`
- final results include all grader fields alongside the raw output

### `001_prompt_evals_complete.ipynb`

The fully assembled evaluation pipeline combining all prior steps:

- dataset with `format` and `solution_criteria` per task
- clean code-only prompting with prefill and stop sequences
- syntax validation and model-based grading with criteria
- results include strengths, weaknesses, reasoning, and a combined score
- prints average score across the dataset

### `001_prompting.ipynb`

This notebook introduces the `PromptEvaluator` class for structured prompt testing:

- defines a `PromptEvaluator` with configurable `max_concurrent_tasks` for parallel test execution
- renders prompt templates with per-test variable substitution
- runs all test cases concurrently using `concurrent.futures`
- scores each result and generates an HTML report (`output.html`) and JSON file (`output.json`)
- uses an athlete meal plan scenario to demonstrate prompt engineering techniques

### `002_prompting_completed.ipynb`

The same `PromptEvaluator` notebook with all cells fully executed, showing rendered reports and grading output.

### `001_tools.ipynb`

This notebook introduces the tool use API:

- defines `get_current_datetime` and `add_duration_to_datetime` as Python functions
- wraps them in `ToolParam` schemas describing their inputs
- manually steps through the tool-calling flow: send request → receive `tool_use` block → run the function → send `tool_result` back

### `001_tools_007.ipynb`

Refactors the helper functions to accept either a plain string or an `anthropic.types.Message` object as content, making them reusable across different call patterns.

### `001_tools_008.ipynb`

Adds automation around tool dispatch:

- `run_tool(tool_name, tool_input)` routes a single tool call to the correct Python function
- `run_tools(message)` extracts all `tool_use` blocks from a response and runs them in sequence
- `run_conversation` loops until the model stops requesting tools

### `001_tools_009.ipynb`

Full multi-tool agentic loop combining both date tools:

- `run_conversation` keeps sending tool results back to Claude until the response contains only text
- demonstrates multi-step reasoning: Claude calls `add_duration_to_datetime` then `get_current_datetime` to answer a natural-language date question

### `003_tool_streaming.ipynb`

Streaming tool use with a `save_article` tool:

- `chat_stream` wraps the streaming API and yields events with optional fine-grained mode
- `save_article` tool saves a generated scholarly article title, abstract, and authors to a dict
- `run_conversation` handles streamed `tool_use` blocks and loops until the model is done

### `003_tool_streaming_completed.ipynb`

The same streaming tool use notebook with all cells fully executed and streaming output visible.

### `005_text_editor_tool.ipynb`

Implements Claude's built-in text editor tool:

- `TextEditorTool` class handles `view`, `create`, `str_replace`, `insert`, and `undo_edit` commands
- maintains a backup directory and undo history per file
- uses the `str_replace_based_edit_tool` schema (`text_editor_20250728`) passed as a tool to the API
- `run_conversation` loops until Claude finishes all edits

### `006_web_search.ipynb`

Starting point for the built-in web search tool, with client setup and message helpers ready to use.

### `006_web_search_complete.ipynb`

Complete web search example using Claude's native search capability:

- defines a `web_search_20250305` schema with a 5-use cap and domain filtering restricted to `nih.gov`
- sends a health query and returns Claude's cited response
- demonstrates how to pass built-in tool schemas alongside the messages API

### `001_chunking.ipynb`

Covers three text chunking strategies for RAG preprocessing:

- `chunk_by_char`: fixed-size windows with configurable size and overlap
- `chunk_by_sentence`: groups sentences into chunks with sentence-level overlap
- `chunk_by_section`: splits a markdown document on `## ` headers

### `002_embeddings.ipynb`

Introduces embedding generation with VoyageAI:

- installs `voyageai` and initializes the client from `.env`
- chunks `report.md` by section
- calls `client.embed` with the `voyage-3-large` model and returns the embedding vector for a chunk

### `003_vectordb.ipynb`

Implements a vector store and a complete dense RAG pipeline:

- `VectorIndex` class stores document vectors and retrieves top-k results by cosine similarity
- full pipeline: chunk `report.md` by section → generate embeddings in bulk → add to `VectorIndex` → embed a user query → retrieve the most relevant chunks

### `004_bm25.ipynb`

Implements sparse keyword retrieval:

- `BM25Index` class with configurable `k1` and `b` parameters and a pluggable tokenizer
- full pipeline: chunk `report.md` by section → add chunks to the index → search with a query string → return ranked results

### `005_hybrid.ipynb`

Combines dense and sparse retrieval into a hybrid search pipeline:

- `Retriever` class wraps a `VectorIndex` and a `BM25Index`, routing documents and queries to both
- `add_documents` bulk-loads chunks into both indexes simultaneously
- search returns fused results ranked by combined dense and sparse scores

## Notes

- `.env` and `.venv/` are ignored by Git and should stay local.
- If you press Enter on an empty line in the chatbot, it asks you to type something instead of sending an invalid request.
- If the API request fails, the notebook prints the error and keeps the conversation history in a valid state.
- `dataset.json` is generated by the eval notebooks and can be regenerated at any time by re-running the dataset cell.
- `output.html` and `output.json` are generated by `001_prompting.ipynb` and can be regenerated by re-running the evaluator cell.
- The RAG notebooks (`002_embeddings.ipynb`, `003_vectordb.ipynb`, `005_hybrid.ipynb`) require a VoyageAI API key. Bulk embedding calls are used where possible to stay within rate limits.
